# OfficeAgent 项目构建计划

根据您的需求和提供的架构图，我将为您构建一个包含前端、后端和工作流引擎的 AI Agent 项目。

## 1. 项目初始化与结构
- **根目录**: `d:\Java开发项目\OfficeAgent`
- **后端 (backend)**: 基于 Spring Boot 3.4.1 + Java 21
- **前端 (frontend)**: 基于 React 18 + TypeScript + Vite
- **数据库**: MySQL 8.0+ (提供 `schema.sql` 初始化脚本)

## 2. 后端开发 (Backend)
### 核心技术栈
- **框架**: Spring Boot 3.4.1, MyBatis-Plus 3.5.5
- **AI 能力**: Spring AI 1.0.0-M5, Spring AI Alibaba 1.0.0-M6.1
- **工具**: FastJSON2, Lombok

### 关键模块实现
1.  **数据模型 (Domain)**:
    -   `Workflow`: 存储工作流元数据。
    -   `WorkflowNode`: 存储节点配置 (ID, 类型, 参数)。
    -   `WorkflowEdge`: 存储节点连接关系。
    -   `ExecutionLog`: 存储执行记录和结果。
2.  **自研 DAG 引擎 (Core Engine)**:
    -   实现 **Kahn 算法** 进行拓扑排序，确定执行顺序。
    -   实现 **DFS** 进行循环检测，防止死锁。
    -   **节点执行器**:
        -   `LLMNodeExecutor`: 调用 Spring AI ChatClient (支持 DeepSeek, 通义千问等)。
        -   `ToolNodeExecutor`: 集成“超拟人音频合成”工具。
        -   `Input/OutputNodeExecutor`: 处理输入输出。
3.  **API 接口**:
    -   工作流 CRUD (保存、加载)。
    -   工作流执行与调试接口。

## 3. 前端开发 (Frontend)
### 核心技术栈
- **框架**: React + TypeScript
- **工作流库**: React Flow (用于画板和节点连线)
- **UI 组件库**: Ant Design

### 关键功能实现
1.  **左侧菜单 (Sidebar)**:
    -   支持拖拽添加“大模型节点” (DeepSeek, 通义千问)。
    -   支持拖拽添加“工具节点” (超拟人音频合成)。
2.  **画板区域 (Canvas)**:
    -   自定义节点 UI (输入、大模型、音频合成、结束)。
    -   节点参数配置面板 (右侧属性栏)。
3.  **调试抽屉 (Debug Drawer)**:
    -   文本输入框 (模拟用户输入)。
    -   执行日志展示。
    -   音频播放器 (播放生成的音频)。

## 4. 实施步骤
1.  **初始化**: 创建前后端项目骨架，配置依赖 (`pom.xml`, `package.json`)。
2.  **数据库设计**: 创建必要的数据库表结构。
3.  **后端核心**: 实现 DAG 引擎和 Spring AI 集成。
4.  **前端实现**: 搭建 React Flow 画布，对接后端接口。
5.  **联调与测试**: 验证“输入 -> 大模型 -> 音频合成 -> 输出”的完整链路。
